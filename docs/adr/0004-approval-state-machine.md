# 0004. 전자결재 상태 머신 + 첨부 DRAFT 한정 + 재상신 폐지

**Status:** Accepted

## Context

전자결재 문서는 여러 상태를 오간다. 상신 이후 문서/첨부를 자유롭게 수정할 수 있으면
결재의 무결성(결재자가 본 내용과 최종본이 다를 위험)이 깨진다. 회수·반려 문서를 그대로
고쳐 재상신하는 흐름도 감사 추적을 흐린다.

## Decision

상태 머신:

```
DRAFT ─상신→ PENDING ─최종승인→ APPROVED
                │
                ├─반려→ REJECTED
                ├─회수→ RECALLED
                └─결재취소→ (이전 단계로 복원)
```

- **첨부 추가/삭제는 DRAFT에서만** 허용한다. 상신 이후에는 잠근다(`ATTACHMENT_LOCKED`).
- **회수/반려 문서는 재상신·수정 불가.** 대신 **"복사하여 새 기안"**으로 내용·결재선을
  복제한 새 DRAFT를 만들어 다시 상신한다(원본 보존).
- 기안함 목록은 DRAFT뿐 아니라 RECALLED/REJECTED도 노출한다(읽기·복사용).

## Consequences

- (+) 결재자가 본 문서와 최종본의 동일성 보장, 감사 추적 명확.
- (+) 원본이 이력으로 남고 재작성본은 별도 문서 → 추적 용이.
- (−) 사용자는 회수/반려 후 "수정 후 재상신" 대신 "복사하여 새 기안"이라는 한 단계를 더 밟는다.
