# 0003. HR 요청 승인자와 전자결재 결재선의 이원화

**Status:** Accepted

## Context

휴가·근무일정·출퇴근 정정 등 HR 요청(`/requests`)은 상신 시 전자결재 문서를 자동
생성한다. "누가 승인하는가"를 두 흐름이 서로 다르게 결정해야 한다는 점이 혼동을 유발했다.

- HR 요청(휴가/요청 메뉴)의 부서 승인자
- 순수 전자결재 문서의 결재선

## Decision

두 승인 주체를 **이원화**한다.

- **HR 요청 부서 승인자** = 요청자 소속 부서의 **조직관리자(ORG_ADMIN 이상)**. 부서에
  없으면 상위 부서로 올라가며 탐색(`resolveDeptApprover`).
- **전자결재 결재선** = `organization.approverId`(결재권자) 기반 결재 규칙.

즉 같은 "승인"이라도 HR 요청 경로와 전자결재 경로의 결정 로직이 다르다.

## Consequences

- (+) HR 요청은 조직 관리 권한 체계를, 전자결재는 결재권자 지정을 각각 자연스럽게 따른다.
- (−) "승인자"라는 용어가 문맥에 따라 다른 대상을 가리켜, 문서/코드에서 명시적 구분이 필요.
- 향후 두 흐름을 통합하는 로드맵이 있다(SYSTEM_DESIGN §5.2.5). 통합 전까지 이원화 유지.
